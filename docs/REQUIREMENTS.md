# 仕様・技術スタックまとめ

## 1. 作るもの
スポットに行き、スマホのカメラでランドマークを撮影し、その写真からオリジナルのスタンプを作成する。
その後、ユーザーがスマホを振りかぶってスタンプを押すことで、実際にスタンプを押したような体験ができるスタンプ集めアプリを作る。

### 方針
- 同じ責務をもつコンポーネントは共通化する

---

## 2. ユーザー体験フロー
1. ユーザーがスポットへ行く
2. スマホのカメラでランドマークを撮影する
3. 撮影後、円形ガイド内にランドマークが収まるようにユーザーが写真を調整する
4. Google Cloud Vision APIでランドマークを判定する
5. 判定に成功したら、撮影写真を円形に切り抜き、スタンプ調に加工する
6. ユーザーがスマホを振り下ろしてスタンプを押す
7. 振りかざすときは軽く振動する
8. 押し込むと強く振動する
9. 斜めに振りかぶった場合は、スタンプが少し傾いて押される
10. 作成されたスタンプをアルバムに保存する

---

## 3. 画面定義

### 画面の種類
- カメラ画面
- 撮影画面
- スタンプを押す画面
- スタンプを押しました画面
- アルバム画面
- マイページ画面
- 通知設定画面
- プライバシー画面
- ヘルプ画面

### 画面遷移
- **起動時**: カメラ画面
- **スタンプ追加フロー**:
  - カメラ（撮影）画面 ➔ スタンプを押す画面 ➔ スタンプを押しました画面

---

## 4. 主要機能

### カメラ画面 - 撮影画面
- ランドマーク撮影
- Vision APIによるランドマーク判定

### カメラ画面 - スタンプを押す画面
- 撮影時の円形ガイド表示
- 撮影後の円形トリミング調整
- 撮影写真のスタンプ画像化
- スマホの加速度・ジャイロを使った振りかぶり判定
- Hapticsによる振動演出
- スタンプ押印アニメーション

### アルバム画面
- スタンプ一覧機能
- コレクションの追加
- コレクションごとの表示

### マイページ画面
- アカウント名
- 最近のコレクション（基本3件 / 「すべて見る」でアルバム画面へ遷移）
- 通知設定
- プライバシー
- ヘルプ

## 5. 技術スタック

### フロントエンド
- **フレームワーク/言語**: React Native, Expo, TypeScript
- **スマホ機能（各種ライブラリ）**:
  - `expo-camera`: カメラ撮影
  - `expo-image-manipulator`: 撮影画像のトリミング・リサイズ
  - `react-native-gesture-handler`: 撮影後のドラッグ・ピンチ操作
  - `react-native-reanimated`: 画像調整やスタンプ押印アニメーション
  - `expo-sensors`: 加速度センサー・ジャイロスコープによる振りかぶり判定
  - `expo-haptics`: 軽い振動・強い振動の演出

### バックエンド
- **言語/フレームワーク**: Python, FastAPI
- **画像処理**: OpenCV
- **処理概要**: アプリから送信された画像を受け取り、Vision APIによるランドマーク判定、OpenCVによるスタンプ調画像加工、画像保存、DB登録を行う。

### 画像認識
- **サービス**: Google Cloud Vision API (Landmark Detection)
- **用途**: 撮影された画像に写っているランドマークを判定するために使用する。

### DB・ストレージ
- **データベース**: Supabase Postgres（スポット情報、ユーザー情報、獲得済みスタンプ情報を保存）
- **ストレージ**: Supabase Storage（加工済みスタンプ画像を保存）
- **方針**: MVPでは元写真は保存せず、加工済みスタンプ画像のみ保存する方針。

### APIデプロイ先候補
- Vercel（※第一候補。デプロイしてだめだったら他を試す）
- Render
- Railway
- Google Cloud Run
- Fly.io
- **選定方針**: APIのデプロイ先はまだ確定せず、無料枠、デモ時の安定性、設定のしやすさを比較して決定する。

---

## 6. 現時点の構成案
React Native / Expoでスマホアプリを作り、Python / FastAPI / OpenCVで画像処理APIを作成する。
ランドマーク判定にはGoogle Cloud Vision APIを使い、スタンプ画像や獲得情報はSupabaseに保存する。
APIのデプロイ先はVercelで、だめそうだったらRender、Railway、Google Cloud Runなどを候補として比較する。
